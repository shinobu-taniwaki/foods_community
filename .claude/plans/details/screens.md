## 1. 全体共通仕様

### 1.1 アプリケーション名

- 表記：**マーケティングCampコミュニティ（MCC）**
- ヘッダー表示：「MCC」または「マーケティングCampコミュニティ」（横幅に応じて切替）
- PWA short_name：`CAMP`

### 1.2 グローバルヘッダー

全認証済み画面で表示。固定（sticky）ヘッダー。

```
┌─────────────────────────────────────────────────────────┐
│ MCC                              🔍   🔔(3)   👤        │
│ マーケティングCampコミュニティ      検索  通知    プロフ │
└─────────────────────────────────────────────────────────┘
```

| 要素 | 配置 | 動作 |
|---|---|---|
| アプリ名 | 左寄せ | タップで `/announcements` へ |
| 🔍 検索 | 右 | タップで `/search` オーバーレイ起動 |
| 🔔 通知 | 右 | タップで `/notifications`。未読件数バッジ（赤丸＋数字、99+ で `99+`） |
| 👤 アバター | 右端 | タップで `/me` へ。絵文字または画像アバター |

- 高さ：56px（モバイル）/ 64px（デスクトップ）
- 背景：クリーム #faf5ed
- 区切り線：オリーブ #5a6b42 を 1px うす表示

### 1.3 グローバルボトムナビゲーション

未ログイン画面・モーダル以外で常時表示。

```
┌──────┬──────┬──────┬──────┐
│ 📣   │ 🏠   │ 📊   │ 👥   │
│お知らせ│掲示板 │データ │ 仲間 │
└──────┴──────┴──────┴──────┘
```

| タブ | パス | アイコン | ラベル |
|---|---|---|---|
| 1 | `/announcements` | 📣 | お知らせ |
| 2 | `/feed` | 🏠 | 掲示板 |
| 3 | `/data` | 📊 | データ |
| 4 | `/members` | 👥 | 仲間 |

- 高さ：64px（safe-area-inset-bottom 含む）
- 選択中タブ：テラコッタ #c05e3f（アイコン＋ラベル）
- 非選択：グレー #6b6b6b
- 背景：クリーム #faf5ed
- 上端区切り：1px オリーブ

### 1.4 配色（プロジェクト共通）

| 用途 | 色 | HEX |
|---|---|---|
| 背景（紙地） | クリーム | `#faf5ed` |
| プライマリ（CTA・強調） | テラコッタ | `#c05e3f` |
| アクセント（タグ・バッジ） | マスタード | `#d9a43d` |
| サブ（リンク・自然色） | オリーブ | `#5a6b42` |
| 本文文字 | ダークブラウン | `#2d2620` |
| 補助文字 | グレーブラウン | `#6b6b6b` |
| エラー | レッドブラウン | `#a33327` |
| 成功 | 深いオリーブ | `#4a5e34` |

### 1.5 タイポグラフィ

- **見出し**：Noto Serif JP（700）。H1 22px / H2 18px / H3 16px
- **本文**：Zen Kaku Gothic Antique（400/500）。基本 16px
- **補助テキスト**：14px、line-height 1.7
- **50代向け配慮**：本文は 16px 以上、ボタンラベルは 16px 以上、タップ領域 48px 以上

### 1.6 レイアウト共通則

- **モバイル幅**：360〜414px が基本想定
- **デスクトップ**：中央 640px カラム、左右に余白
- **角丸**：14px（カード・モーダル・ボタン）
- **影**：原則使わない、必要時のみ `0 1px 2px rgba(0,0,0,0.04)` 程度
- **コンテンツ余白**：左右 16px、要素間 16px

### 1.7 共通インタラクション

- **タップフィードバック**：押下時に 100ms で 0.95 倍スケール＋うす陰
- **遷移**：ページ遷移は即時、ローダーは 300ms 以上の処理にのみ表示
- **無限スクロール**：掲示板・通知・お知らせは下端到達時に追加ロード（20件/ページ）
- **プルツーリフレッシュ**：MVP では非対応（v0.2 で検討）

### 1.8 共通モーダル仕様

- 暗いオーバーレイ（rgba(0,0,0,0.4)）背景
- ボトムシート（モバイル）/ センターモーダル（≥640px）
- 角丸 14px、上部に灰色のグラブハンドル（モバイル）
- 閉じる手段：×ボタン、背景タップ、Esc キー（デスクトップ）

### 1.9 PWA / メタ情報

- アプリ短縮名：CAMP
- 起動色：`#faf5ed`
- テーマ色：`#c05e3f`
- iOS Safari 用 apple-touch-icon を別途用意

### 1.10 アクセシビリティ基本方針（50代対応）

| 項目 | 基準 |
|---|---|
| 本文最小サイズ | 16px |
| タップ領域 | 48×48px 以上 |
| コントラスト比 | テキスト 4.5:1 以上、大きなテキスト 3:1 以上 |
| フォーカスリング | 2px テラコッタ、外周にうす影 |
| 文字色 | 純黒ではなくダークブラウン（紙との親和性） |
| アイコンのみのボタン | 必ず aria-label を付与 |
| エラーメッセージ | 平易な日本語、解決手順を添える |

**目的・ユーザー価値**：登録済みユーザーが MCC にログインする。

**アクセスできる人**：未認証ユーザー。

│  ─────── または ────────         │

| 認証成功 | `/announcements` へ遷移 |

- 👁 アイコン：パスワード表示/非表示

**API / Server Action**
- `signInWithPassword({ email, password })`
- `signInWithOtp({ email })`（Magic Link）
- `signInWithOAuth({ provider: 'google' })`
- `resetPasswordForEmail({ email })`

| アカウント退会済 | 「このアカウントは退会済みです」 |
| ネットワーク | 「接続できませんでした。電波の良い場所でお試しください」 |

- `verifyInvitationToken({ token })`
- `acceptInvitationWithGoogle({ token })`（OAuth コールバック後）
- `acceptInvitationWithPassword({ token, password })`
- トランザクション内：auth.users 作成、profiles INSERT、notification_preferences INSERT、invitations.accepted_at 更新

| パスワード弱い | 「英字と数字を両方含めてください」 |

| トランザクション失敗 | 「登録に失敗しました。しばらく経ってからお試しください」 |

### 4.5 エラーメッセージ方針

**3原則**：
1. 何が起きたかを平易な日本語で伝える
2. ユーザーが次に何をすべきかを示す
3. 専門用語を避け、運営に責任を寄せる表現を選ぶ

**避ける表現**
- 「Internal Server Error」「500」「失敗しました（コード: XXXX）」
- 「不正なリクエスト」「Validation Error」
- 専門用語（トークン、セッション、CSRF など）

**推奨表現**
- 「読み込みに失敗しました [再読み込み]」
- 「保存できませんでした。少し待ってから [再試行]」
- 「電波の良い場所でお試しください」
- 「お時間をおいて再度お試しください。解消しない場合は [お問い合わせ]」
- 「申し訳ありません、一時的にご利用いただけません」

**トースト表示**
- 成功：3秒、左下から
- エラー：5秒、左下から（再試行ボタン付きの場合は手動閉じるまで）

## 6. 補足：実装時の注意点

### 6.1 trial プランの UX

- ロック機能タップ時は**エラーで止めず**、`/upgrade` へ穏やかに誘導
- ぼかし表示＋「プランを変更」CTA で「見えるけど触れない」を表現
- イライラさせない言葉選び：「アップグレードしてください」ではなく「もっと使うには」

### 6.2 admin 画面のレスポンシブ

- admin はデスクトップ利用想定が多いが、しのぶさんはモバイル利用もあり得る
- 一覧テーブルはモバイルでカード形式に切替（CSS でレスポンシブ）

### 6.3 退会者・停止者の表示

- メンバー画面：「（退会したメンバー）」「（停止中のメンバー）」表記でコメント等を残す
- admin 画面：本人名で表示、ステータスバッジ付き

### 6.4 リアルタイム更新

- 掲示板トップ：新規投稿のバナー通知
- 通知ベル：未読バッジを Realtime で更新
- お知らせ詳細：新規コメントを末尾追加
- 全機能で「上書き型」ではなく「追加型」を使い、ユーザー操作中の入力を消さない

### 6.5 Google フォーム連携の共通パターン

- 共通コンポーネント `ExternalFormLink` を使用
- prefill：氏名・メール・関連情報をクエリで渡す
- target="_blank" rel="noopener noreferrer"
- フォームから戻ってきても画面状態を保持

### 6.6 画像表示の最適化

- リスト表示時はサムネのみ（200×200px 程度）
- 詳細画面はライトボックスで元画像
- Next.js `<Image>` で自動最適化＋遅延読み込み
